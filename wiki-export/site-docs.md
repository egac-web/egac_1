Documentation for the site needs to be up to date, maintained and accurate. The following are a minimum standard expected.

The site must have a readme.md file that is maintained and remain up to date.

The site must have a change.md file to record all project changes and remain up to date.

There must be a log of issues and fixes that will be maintained throughout the build.

## Additional Documentation Requirements

- All major features and integrations (e.g., Directus, Cloudflare, analytics, forms) must be documented in dedicated markdown files.
- Each page/component should have a brief description of its purpose and usage, either in the README or a separate file.
- Environment variables and deployment steps must be clearly listed and updated as needed.
- Any scripts, automation, or CI/CD setup should be described in a scripts.md or ci-cd.md file.
- Accessibility and design decisions should be documented (see colors-and-background.md).
- Social media and contact details should be kept current in their respective files.

## Recommended File List

- readme.md
- change.md
- issues-log.md
- project-structure.md
- directus-integration.md
- colors-and-background.md
- enquiry_form.md
- social-media.md
- site-docs.md
- scripts.md (if needed)
- ci-cd.md (if needed)
- variables.md

- dummy-records.md (sample Directus records for testing)

## Maintenance

- Documentation should be reviewed and updated with every major change or deployment.
- Outdated or deprecated features must be marked clearly in the docs.
- Contributors should add notes for any decisions or trade-offs made during development.
